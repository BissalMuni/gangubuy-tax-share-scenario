# Specification

## Project: share-scenario

지방세법령 개정 내용을 기반으로 세금정보 제공 유튜브 영상 시나리오를 제작·공유·협업하는 플랫폼.

## 사용자 역할

| 역할 | 설명 |
|------|------|
| **관리자** | 시나리오 import, 트리 구조 관리, 상태 변경, 전체 관리 |
| **제작자** | 시나리오 열람·수정, 콘티 업로드, 콘티 열람·평가 |

- 로그인 불요 (인증 없이 사용)
- 관리자 기능은 별도 관리 페이지 또는 비밀번호 보호로 구분

## User Stories

### US-1: 트리 네비게이션으로 시나리오 탐색

> As a 제작자, I want to 트리 구조로 시나리오 목록을 탐색하고 싶다, So that 원하는 법령/세목의 시나리오를 빠르게 찾을 수 있다.

- 드롭다운으로 분류 기준(법령별, 납세자 유불리 등) 전환 가능
- 트리 노드를 펼치고 접을 수 있다
- 리프 노드 클릭 시 해당 시나리오 상세 페이지로 이동
- 각 시나리오의 현재 상태(draft/revision/production/complete)가 트리에 표시

### US-2: 시나리오 상세 조회

> As a 제작자, I want to 시나리오의 원본 PDF와 스크립트를 함께 볼 수 있다, So that 법령 개정 내용과 영상 시나리오를 대조할 수 있다.

- 원본 PDF 페이지 뷰어 (toc 폴더의 PDF)
- 시나리오 메타정보 표시 (제목, 법률근거, 시행일, 영상길이 등)
- 나레이션 스크립트(script.md) 표시
- 장면별 구성(scenario.json의 scenes) 표시

### US-3: 시나리오 웹 수정

> As a 제작자, I want to 웹에서 시나리오 내용을 수정할 수 있다, So that 별도 도구 없이 바로 개선할 수 있다.

- 스크립트(나레이션) 텍스트 편집
- 장면별 구성 수정 (비주얼, 나레이션, 텍스트 오버레이 등)
- 메타정보 수정 (제목, 영상길이 등)
- 수정 이력 확인 가능

### US-4: 콘티(Storyboard) 업로드 및 공유

> As a 제작자, I want to 시나리오별로 콘티를 업로드하고 다른 사람의 콘티를 볼 수 있다, So that 다양한 영상 제작 아이디어를 공유할 수 있다.

- 시나리오별로 여러 명이 각자 콘티 업로드 가능
- 지원 형식: 이미지(JPG, PNG), PDF
- 콘티 목록 갤러리 형태로 조회
- 콘티에 대한 간단한 평가 가능 (기록 저장 불필요, 휘발성)

### US-5: 시나리오 상태 관리

> As a 관리자, I want to 각 시나리오의 제작 단계를 관리할 수 있다, So that 전체 영상 제작 진행 현황을 파악할 수 있다.

- 시나리오 상태 변경: draft → revision → production → complete
- 대시보드에서 전체 시나리오의 상태별 현황 조회
- 상태별 필터링

### US-6: 시나리오 데이터 초기 Import

> As a 관리자, I want to revised 폴더의 데이터를 일괄 import할 수 있다, So that 기존 작업 결과를 플랫폼에 올릴 수 있다.

- toc/ 폴더의 PDF를 Supabase Storage에 업로드
- scenarios/ 폴더의 scenario.json + script.md를 DB에 저장
- 폴더 구조를 기반으로 초기 트리 JSON(tree-by-law.json) 자동 생성

### US-7: 트리 분류 관리

> As a 관리자, I want to 새로운 분류 기준의 트리를 추가할 수 있다, So that 다양한 관점으로 시나리오를 재분류할 수 있다.

- 트리 JSON 파일 추가/수정
- 기존 시나리오를 새로운 트리 노드에 매핑
- [NEEDS CLARIFICATION] 트리 편집을 웹 UI에서 하는가, JSON 파일 직접 편집인가?

## 성공 기준

1. revised 폴더의 전체 시나리오(약 90개)가 import되어 트리에서 탐색 가능
2. 법령 기준 트리와 최소 1개 이상의 대체 분류 트리가 동시 제공
3. 시나리오 스크립트를 웹에서 수정하고 저장할 수 있음
4. 시나리오별 콘티를 업로드하고 갤러리로 조회 가능
5. 전체 시나리오의 제작 단계 현황을 대시보드에서 한눈에 파악 가능

## [NEEDS CLARIFICATION]

1. **트리 편집 방식**: 관리자가 웹 UI에서 트리 구조를 편집하는가, 아니면 JSON 파일을 직접 수정하는가?
2. **수정 권한**: 모든 제작자가 시나리오를 수정 가능한가, 아니면 특정 시나리오에 대한 담당자 지정이 필요한가?
3. **콘티 평가 방식**: 휘발성 평가의 구체적 형태는? (좋아요 버튼, 별점 등 — 새로고침 시 초기화)
